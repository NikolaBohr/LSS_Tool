Parser – Negative Frames & Multiple Numeric Segments

▶ parser.py changed

Regex pattern update

# Pattern: capture everything up to the last number before extension
# Groups: (base)(frame with optional negative sign)(extension including dot)
_SEQUENCE_PATTERN = re.compile(
    r"^(.*?)(-?\d+)(\.[^.]+)$"
)

Two key changes from baseline:

  • (.*?) — Non-greedy base capture. Matches as little as possible, so the last numeric segment becomes the frame number. This allows filenames with multiple numbers in the base (e.g. shot_01_v002.)
  • (-?\d+) — Optional minus sign before digits. Enables negative frame numbers like -0010
▶ parser.py new

Multiple numeric segments – how it works

The non-greedy (.*?) ensures the regex always captures the last number as the frame:

Input filenamebaseframepaddingextension
shot_01_v002.0001.exr shot_01_v002. 1 4 .exr
render_v03.100.png render_v03. 100 0 .png
foo.0004.exr foo. 4 4 .exr

The numbers 01, 002, v03 in the base are preserved as part of the grouping key — only the last number before the extension is treated as the frame.

▶ parser.py new

Negative frame numbers – padding detection

# For negative numbers, check the digit portion after the minus sign
digits = frame_str.lstrip("-")
if len(digits) > 1 and digits[0] == "0":
    padding = len(digits)
else:
    padding = 0

anim.-5.exr -5 5 -5 0

Padding is determined by the digit portion only, ignoring the minus sign. This means -0010 has padding=4 (the 0010 part), matching positive frames with the same zero-padding.

Symlink following – -L / --follow-symlinks

# Check if it's a file (respecting symlink preference)
if entry.is_file(follow_symlinks=follow_symlinks):
    files.append(entry.name)
# Also include symlinks to files when following symlinks
elif follow_symlinks and entry.is_symlink():
    try:
        if entry.resolve().is_file():
            files.append(entry.name)
    except (OSError, ValueError):
        pass  # Broken symlink - skip

Without -L, symlinks are not resolved. With -L, symlinks pointing to files are included; broken symlinks are silently skipped.

▶ cli.py new

Permission error handling – per-file recovery

# Directory-level errors
try:
    entries = list(path.iterdir())
except PermissionError:
    return [], [f"lss: cannot open directory '{path}': Permission denied"]

# Per-file errors (continue processing)
for entry in entries:
    try:
        if entry.is_file(...):
            files.append(entry.name)
    except PermissionError:
        errors.append(f"lss: cannot access '{entry.name}': Permission denied")
    except OSError as e:
        errors.append(f"lss: cannot access '{entry.name}': {e.strerror}")

Error flow in main():

# Report errors to stderr but continue
for error in errors:
    print(error, file=sys.stderr)

# Exit with error only if we got nothing
if not filenames and errors:
    return 1

# Non-zero exit if there were any errors (like ls does)
return 1 if errors else 0

This mirrors ls behavior: report per-file permission errors to stderr, but still process and display the files that can be read. Exit code reflects whether any errors occurred.

Sorting Options – New Feature Deep Dive

▶ grouper.py new

SortOrder class

class SortOrder:
    """Sort order options for group_files output."""
    NAME      = "name"       # Alphabetical by name (default)
    FRAME     = "frame"      # By first frame number
    COUNT     = "count"      # By number of frames in sequence
    EXTENSION = "extension"  # By file extension

Plain class with string constants. Used as enum-like values for argparse choices and the sort_key function.

▶ grouper.py changed

group_files() – new signature & sort logic

def group_files(
    filenames: list[str],
    sort_by: str = SortOrder.NAME,
    reverse: bool = False,
) -> list[FileSequence | str]:
    # ... grouping logic unchanged ...

    # Build sort key based on sort_by option
    def sort_key(item: FileSequence | str) -> tuple:
        if isinstance(item, FileSequence):
            name      = item.format()
            first_frame = min(item.frames) if item.frames else 0
            count     = len(item.frames)
            extension = item.extension.lower()
        else:
            name      = item
            first_frame = 0
            count     = 1
            extension = item.rsplit(".", 1)[-1].lower() if "." in item else ""

        if sort_by == SortOrder.FRAME:
            return (first_frame, name)
        elif sort_by == SortOrder.COUNT:
            return (count, name)
        elif sort_by == SortOrder.EXTENSION:
            return (extension, name)
        else:  # Default to NAME
            return (name,)

    result.sort(key=sort_key, reverse=reverse)

Key design decisions:

  • Sort key always returns a tuple – secondary sort is always by name for stability
  • Plain strings (non-sequence files) get sensible defaults: frame=0, count=1
  • reverse passed directly to list.sort() – no manual inversion
  • The old baseline had a simple result.sort(key=lambda x: ...) by name only

Color Output – New Feature Deep Dive

▶ cli.py new

Colorama import strategy

# Graceful import with fallback
try:
    import colorama
    from colorama import Fore, Style
    COLORAMA_AVAILABLE = True
except ImportError:
    COLORAMA_AVAILABLE = False
    # Dummy replacements — empty strings
    class Fore:
        CYAN = GREEN = YELLOW = RED = RESET = ""
    class Style:
        BRIGHT = RESET_ALL = ""

Key: colorama is an optional dependency (declared in [project.optional-dependencies]). When absent, Fore.CYAN etc. resolve to "", so the formatting f-strings produce plain text.

format_output() – color decomposition

def format_output(item: FileSequence | str, use_color: bool = False) -> str:
    if isinstance(item, FileSequence):
        formatted = item.format()
        if use_color:
            # Decompose into colored segments:
            #   CYAN      base       "foo."
            #   BRIGHT    range#pad  "0-4#4"
            #   GREEN     extension  ".exr"
            return (
                f"{Fore.CYAN}{item.base}{Style.RESET_ALL}"
                f"{Style.BRIGHT}{item._format_frame_range(sorted(item.frames))}"
                f"#{item.padding if item.padding > 0 else ''}{Style.RESET_ALL}"
                f"{Fore.GREEN}{item.extension}{Style.RESET_ALL}"
            )
        return formatted
    else:
        if use_color:
            return f"{Fore.YELLOW}{item}{Style.RESET_ALL}"
        return item
